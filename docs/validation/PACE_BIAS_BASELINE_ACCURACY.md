## 騎手フラグの効果検証（新規追加）

`bias_position_harmony` 上位5頭を選出した後、騎手フラグ別に精度を比較。

| 条件 | カバー率 | 候補精度 | レース数 | 全体比 |
|---|---|---|---|---|
| [A] 全体（再掲） | 82.3% | 69.5% | 4,941 | 基準 |
| [B] 上位5頭に `leading_flag=1` が 1頭以上 | **84.6%** | **72.4%** | 2,327 | **+2.3% / +2.9%** |
| [C] 上位5頭に `leading_flag=1` が ゼロ | 80.3% | 67.0% | 2,614 | -2.0% / -2.5% |
| [D] `weight_reduction_flag=1` を除外した上位5頭 | **83.2%** | **71.7%** | 4,925 | **+0.9% / +2.2%** |

### 解釈

**[B] vs [C] の差が 4.3%（カバー率）/ 5.4%（候補精度）**:

> リーディング騎手が上位5頭に含まれるレースは、そうでないレースより明確に高精度。**騎手の質が `harmony` フィルターの有効性を高める**ことが確認できた。

**[D] 減量騎手除外の効果**:

> 減量騎手（kinryo 相対判定で軽い）を候補から除外すると、カバー率 +0.9%、候補精度 +2.2% の改善。ただしレース数が 4941→4925（16 レース減）なので効果の絶対値は小さい。

### 活用方針

```
harmony 上位5頭フィルター の正しい使い方:
  基本: harmony スコアで上位5頭を選出
  +α:  その中に jockey_leading_flag=1 の馬がいれば信頼度 UP
  除外: weight_reduction_flag=1 の馬は候補から落とすと精度 +0.9%
```

## 総括

### 騎手フラグ修正で改善したか？

**YES（騎手フラグの有効性が確認できた） / NO（フィルター全体精度は変化なし）**

| 観点 | 判定 | 詳細 |
|---|---|---|
| `weight_reduction_flag` の正確性 | **大幅改善** | 67.5% → 20.7%（kinryo 相対判定で適正化） |
| `jockey_leading_flag` の機能確認 | **機能確認** | リーディング有無で 4.3% の差を確認 |
| フィルター全体精度（harmony スコア） | **変化なし** | 修正はフラグの精度向上であり harmony の再計算には影響しない |
| 騎手フラグを組み合わせた追加精度 | **+0.9〜+2.3%** | leading=1 絞り込み・減量除外で改善 |

### フィルターとして自然確率を有意に上回るか？

**YES — 全セグメントで一貫して +4〜5% の優位性を維持。**

| 比較 | 判定 |
|---|---|
| harmony vs ランダム | **上回る** (+4〜5%、全セグメントで一貫） |
| harmony vs 人気1〜5番 | **下回る** (82.3% vs 98.4%、差 -16%) |
| harmony ランク別単調性 | **確認** (1位 26.7% → 6位以下 19.3%、+7.4%) |
| 騎手要素との組み合わせ効果 | **確認** (leading_flag=1 で +2.3%、excluding reduction で +0.9%) |

### 位置づけと推奨活用

```
harmony フィルターの適切な位置づけ:
  - 単独フィルター: 人気より弱い（82.3% vs 98.4%）
  - 人気と独立した別軸: 脚質×バイアス整合性 → 穴馬検出に有効
  - 組み合わせ効果: 人気 + harmony + 騎手フラグで精度向上期待

推奨実装:
  1. 全出走馬を harmony で順位付け
  2. 人気上位 (1〜8番人気) に絞ってから harmony 上位5頭を選出
  3. leading_flag=1 の馬を優先（同スコアの場合）
  4. weight_reduction_flag=1 の馬を除外（候補が足りる場合）
```
