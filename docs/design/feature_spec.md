# 競馬予測AI Webサービス版 特徴量・システム仕様書 (v1.7)

## 1. システムアーキテクチャの基本方針
* **既存システムの保護:** 既存のJRA-VANデータ取得パイプラインには一切干渉しない。
* **完全な疎結合（プラグイン化）:** `web_service/` 配下で管理。
* **絶対死守事項：サブモデル分割戦略（スタッキング）:** 展開（ペース・位置取り）などの複雑な事象は単一モデルに混ぜず、必ず専用の「展開予測サブモデル」を構築して段階的に予測させる。

## 2. 特徴量設計のコンセプト：「点」から「線」へ
単発の成績から、馬のライフサイクルや適性の変化（線）、調教プロセスの推移、および「展開の巡り合わせ」を捉える。

## 3. オフライン・フィーチャーストア（事前計算DB基盤）
データリークを物理的に防ぐため、前日時点のスナップショットを保持する。

### 3.1 外的要因と血統ポテンシャル
* 騎手・調教師の成績、種牡馬の四大能力（スピード/スタミナ/パワー/操縦性）、性別適性、成長曲線。

### 3.2 コース特性プロファイル（CourseProfileStore）
競馬場×距離ごとの「極端なバイアス」を人間がハードコードするのではなく、データとしてAIに認識させる。
* **保持指標:** 当該コースにおける「枠順（1〜8枠）ごとの勝率シフト」「脚質（逃げ・先行・差し・追込）ごとの勝率シフト」。

### 3.3 馬レーティング（HorseRatingStore）
Elo/TrueSkillの概念を用いた馬の絶対能力値。レースごとに変動し、日次バッチで更新。
* **保持指標:** 馬自身の出走前夜時点のレーティングスコア。
* **効果:** 過去レースの「出走馬の平均レート」を算出することで、データリークなしに「伝説の新馬戦（超ハイレベル戦）」を客観的数値としてAIに認識させる。

### 3.4 黄金コンビ指標（SynergyStore） ※v1.6追加
調教師×騎手の組み合わせによる「陣営の勝負気配・本気度」を数値化する。
* **保持指標:** コンビ結成時の勝率・複勝率と、それぞれの単独期待値からの「上振れ幅（Synergy Shift値）」。

## 4. 展開とコンテキスト（スタッキングアーキテクチャ）
レースの展開と、馬の「得意な形」への合致度をサブモデルを用いて評価する。

### 4.1 展開予測サブモデル（Pace & Position Model）
1. **レースペース予測確率:** ハイペース(H) / ミドル(M) / スロー(S) になる確率。
2. **想定ポジション（位置取り）:** 各馬の最初のコーナー通過順位の予測値。

### 4.2 得意形マッチング（Suitability Score）
* **ポジション合致度:** 「過去勝利時の平均通過順位」と「予測ポジション」の差分。
* **展開の恵まれ度:** 「予測ペース」に対する「過去の同ペース戦での成績シフト値」を加味。

## 5. 陣営の本気度と調子（調教評価パイプライン） ※v1.6追加
人間の主観（ルールベースの点数や「馬なり/一杯」の記号）を完全廃止し、純粋な客観データからナマの調子を判定する。

### 5.1 調教の客観的評価（TrainingZScoreプラグイン）
旧パイプラインの計算ロジックを独立プラグインとして再設計。
* 馬場や時間帯のノイズを排除した「同日・同コースにおけるタイムのZ-score（偏差値）」。
* 終いのキレを測る「加速ラップ（lap_2 - lap_1）」。

### 5.2 調教プロセスの得意形マッチング（ConditionMatchScorer）
直前の1本だけでなく、前走からの「プロセス（点ではなく線）」を評価する。
* **評価対象:** 乗り込み本数、1週前と最終追い切りの時計の推移、坂路とウッドの併用割合。
* **マッチング:** 今回の「一連の調教プロセス」が、その馬の「過去の必勝調教パターン」とどれだけ合致しているかを距離（差分）としてスコア化する。

## 6. 後天的能力（実績）のベースラインと補正タイム
表面的な着順・タイムに騙されないための高度な補正処理。

### 6.1 ベース能力と斤量
* **着差・斤量:** 前走着差、前走タイム偏差値、レース階層（クラス）、平均斤量との差、前走比。

### 6.2 補正タイムと上位クラス比較
* その日の「馬場差（基準タイムとの乖離）」を算出し、走破タイムを補正。
* さらに「上位クラスの基準タイム」と比較することで、昇級戦でも通用するか（潜在能力）を数値化する。

## 7. XAI（説明可能なAI）と予測根拠の視覚化
AIの推論をブラックボックス化せず、ユーザーが納得できる「評価ポイント」をUI上に提示する。
* **表示例:** 「【調教パターン合致】中間にウッドで負荷をかけ、最終は坂路で軽く流すという、この馬の過去の必勝調教プロセスを完全に再現しています」「【勝負気配アリ】〇〇調教師と〇〇騎手の黄金コンビ。単独時より勝率が15%跳ね上がる勝負の布陣です」

### 8.2 pace_features_v4 + pace_simulation_v1（pace_v2 サブモデル向け / 19 特徴量）

> **⚠️ 永久除外リスト（データリーク）**  
> `pace_index` / `lap_variance` / `lap_std` / `pace_type` / `zen_3f` / `go_3f` は  
> すべて当走後に確定する事後データ。コードから削除済み。絶対に再追加しないこと。

#### コーナー位置の正規化方式
`norm = (corner順位 - 1) / (出走頭数 - 1)` → 0.0(先頭) 〜 1.0(最後尾)

**頭数正規化脚質（pace_features_v4 / 4 特徴量 / 全距離）**

| カラム名 | 説明 | NaN 補完値 |
|---|---|---|
| `avg_c1_norm_5` | 第1コーナー正規化位置の直近5走平均 | 0.5（中団相当） |
| `avg_c4_norm_5` | 第4コーナー正規化位置の直近5走平均 | 0.5 |
| `avg_pos_advance_norm_5` | (C4位置 − 着順) の平均。正値=追い込み気質 | 0.0 |
| `running_style_std_norm_5` | C1正規化位置の標準偏差（脚質ブレ幅） | 0.15 |

**距離区分別（サフィックス: `_sprint` / `_mile` / `_mid` / `_long` / 12 特徴量）**

| 区分 | 距離帯 | カラム群（3列） |
|---|---|---|
| sprint | ≤ 1400m | `avg_c1_norm_5_sprint`, `avg_c4_norm_5_sprint`, `avg_pos_advance_norm_5_sprint` |
| mile | 1500〜1800m | `avg_c1_norm_5_mile`, `avg_c4_norm_5_mile`, `avg_pos_advance_norm_5_mile` |
| mid | 1900〜2200m | `avg_c1_norm_5_mid`, `avg_c4_norm_5_mid`, `avg_pos_advance_norm_5_mid` |
| long | ≥ 2300m | `avg_c1_norm_5_long`, `avg_c4_norm_5_long`, `avg_pos_advance_norm_5_long` |

距離帯出走歴なしの場合 NaN 補完 = **0.5**

**展開シミュレーション（pace_simulation_v1 / 3 特徴量 / 完全事前データ）**

`avg_first_corner_norm_5` + `umaban`（今回枠順）のみから計算。事後データ一切不使用。  
> ⚠️ **v1.8 変更:** 入力を `avg_c1_norm_5` → `avg_first_corner_norm_5` に変更。  
> 理由: 1400m 以下は corner_1 が常に 0 のため。c1→c2→c3→c4 優先で最初のコーナーを動的取得。  
> pace_v2 サブモデルの再学習が必要（`docs/PROGRESS.md` 参照）。

| カラム名 | 説明 | 範囲 |
|---|---|---|
| `predicted_position_norm` | 今回レースの推定ポジション（0=先頭 / 1=最後尾）。枠順補正ランキング方式 | [0, 1] |
| `predicted_field_pace` | フィールド全体の推定ペース指数（先行馬比率 + 平均積極性） | [0, 1] |
| `pace_harmony_pre` | ペース合致度 = (1−pos)×(1−pace) + pos×pace。前づけ×スロー or 差し×ハイ = 1.0 | [0, 1] |

ソース: `src/features/pace_simulation_v1.py`  
**データソース:** `race_entries.corner_1/corner_4/kohan_3f/umaban` + `races.distance/track_code`  
**リーク防止:** shift(1) + rolling — 当走を除く直近5走の統計

### 8.3 NaN 補完のドメイン根拠

| 補完先 | 補完値 | 根拠 |
|---|---|---|
| 着順系（prev1_rank 等） | 18.0 | フルゲート18頭の最下位 = キャリアなし馬の保守的評価 |
| 勝率・複勝率系 | 0.0 | 実績ゼロ = 完全未知 |
| グレード | 1.0 | 未勝利クラス相当 |
| コーナー正規化位置 | 0.5 | 中団（先行でも追込でもない） |
| 順位変化 (pos_advance) | 0.0 | 動きなし（ポジション維持） |
| 脚質ブレ幅 | 0.15 | 実測データから算出した代表的な標準偏差値 |
| 上がり順位 | 8.0 | 典型的な中位フィニッシュ |
| 上がり順位 std | 3.0 | 典型的なばらつき |

補完は **SHAP 値の説明性を保つため** — `NaN のまま LightGBM に渡すと内部で 0 相当に扱われ、ドメイン上の意味と乖離する。
